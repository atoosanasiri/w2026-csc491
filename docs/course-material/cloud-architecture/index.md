## Course Sections

This lecture is organized into seven comprehensive sections:

1. [Section 1: Thinking Like an Architect](section-1-thinking-like-architect.md) - Learn the transition from coding to system design, understanding the shift from building features to architecting services.

2. [Section 2: Pre-Selection Analysis](section-2-pre-selection-analysis.md) - Explore tenancy models, infrastructure responsibility, and the critical decision between managed services and self-hosted solutions.

3. [Section 3: Advanced Architectural Patterns](section-3-advanced-patterns.md) - Master API Gateway patterns, static/dynamic separation, event-driven architecture, and edge computing.

4. [Section 4: Operational Excellence in the Cloud](section-4-operational-excellence.md) - Understand reliability, self-healing systems, security, scalability, and observability.

5. [Section 5: Automation is Not Optional](section-5-automation.md) - Discover why automation is foundational, covering CI/CD, Infrastructure as Code, secret management, and deployment strategies.

6. [Section 6: The MVP Architect's Dilemma](section-6-mvp-dilemma.md) - Navigate the balance between tactical execution and architectural foundations, learning what to defer and what to build now.

7. [Section 7: Suggested Reading & Bibliography](section-7-reading-bibliography.md) - Access curated resources, essential texts, and references for deeper exploration of cloud architecture topics.
